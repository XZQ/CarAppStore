## 3. Java / Kotlin 版本统一
当前所有 Android module 已统一为：

```kotlin
compileOptions {
    sourceCompatibility = JavaVersion.VERSION_11
    targetCompatibility = JavaVersion.VERSION_11
}

kotlinOptions {
    jvmTarget = "11"
}
```

目的：

- 保持全工程目标版本一致
- 避免个别 module 漂移到 17
- 为后续真实编译验收提供统一基线

---

## 4. 依赖集中管理
当前三方依赖版本已经集中到：

- `gradle/libs.versions.toml`

各 module 的 `build.gradle.kts` 现已改为通过 version catalog 引用依赖，例如：

- `libs.androidx.core.ktx`
- `libs.androidx.appcompat`
- `libs.google.material`
- `libs.androidx.fragment.ktx`
- `libs.kotlinx.coroutines.android`

这样做的价值：

- 版本号只维护一处
- 批量升级成本更低
- 避免多 module 下依赖版本漂移

---

## 5. 字符串资源治理
本轮继续清理不可读、不可维护的资源 id。

以开发设置页下载环境文案为例，已经改为可读 id：

- `ui_download_environment_current_format`
- `ui_download_environment_hint_dev`
- `ui_download_environment_hint_test`
- `ui_download_environment_hint_prod`
- `ui_download_environment_switch_dev`
- `ui_download_environment_switch_test`
- `ui_download_environment_switch_prod`

原则：

- 新增字符串必须使用可读、可维护的资源名
- 不再继续引入异常长的转义式 id
