# 25. 真机回归测试清单

> 最后更新：2026-04-20
> 本文档面向 QA 和开发人员，用于在真实车机设备上逐项执行回归验证。

## 1. 测试前置条件

### 1.1 设备信息记录

每轮测试开始前，先填写以下模板，确保结果可追溯。

```
| 测试日期     |                               |
| 测试人员     |                               |
| 构建版本号   |                               |
```

### 1.2 测试环境确认

在开始测试前逐项确认：

- [ ] APK 已通过 `./gradlew app:assemble` 构建并安装到设备
- [ ] 应用可正常冷启动，无闪退
- [ ] 已选择正确的下载环境（开发/测试/生产），通过开发者设置确认
- [ ] 目录源可正常加载（网络连通或资源目录回退正常）
- [ ] 设备存储空间充足（>100MB 可用）
- [ ] 日志工具已就绪（adb logcat 或车机调试口）
- [ ] 网络抓包工具已就绪（如需验证 HTTP 请求）

## 2. 下载链路回归

### 2.1 基本下载

- [ ] **正常下载完成（小文件 <10MB）**：点击下载后进度条平滑推进，状态从"下载中"变为"下载完成"，文件完整
- [ ] **正常下载完成（大文件 >100MB）**：长时间下载过程中进度条持续更新，不出现 ANR 或卡死，下载完成通知正常

### 2.2 下载控制

- [ ] **下载暂停并恢复**：暂停后进度条停止，恢复后从断点继续，进度值衔接无回退
- [ ] **下载取消并确认清理**：取消后任务从列表移除，临时文件和元数据被清理，存储空间恢复

### 2.3 网络异常

- [ ] **网络断开后下载失败并自动标记**：断网后任务状态变为"下载失败"，错误信息可读
- [ ] **网络恢复后重试下载**：重新点击下载后从断点续传，不从头开始
- [ ] **冷启动后恢复上次下载进度**：下载中途强杀进程，重新打开后任务仍存在且进度正确，可继续下载

### 2.4 下载完整性

- [ ] **Range 分段下载正确合并**：大文件分段下载后合并结果与源文件一致，安装包可正常解析
- [ ] **下载文件校验（SHA-256 匹配）**：校验通过时正常进入安装流程
- [ ] **下载文件校验（SHA-256 不匹配）**：校验失败时标记错误，提示用户重试，不进入安装

### 2.5 边界条件

- [ ] **存储空间不足时拒绝下载**：可用空间低于 APK 大小时拒绝发起下载，提示"存储空间不足"
- [ ] **同一应用重复点击下载**：不产生重复下载任务，已有任务继续执行

## 3. 安装链路回归

### 3.1 正常安装

- [ ] **正常安装完成**：下载完成后自动或手动触发安装，安装进度正常，最终显示"安装成功"
- [ ] **系统确认页弹出并用户确认**：PackageInstaller 确认页正常弹出，点击确认后安装继续
- [ ] **安装成功后下载任务自动清理**：安装成功后，对应下载任务和临时文件被自动清理

### 3.2 安装中断

- [ ] **用户拒绝安装确认**：系统确认页点击取消，应用状态回退到"可安装"，可重新触发
- [ ] **APK 损坏时安装失败并正确回退**：安装失败后状态变为"安装失败"，错误信息明确，可重试
- [ ] **安装中断后恢复（会话持久化）**：安装过程中强杀进程，重新打开后安装会话可恢复或重新发起

### 3.3 升级安装

- [ ] **升级安装（覆盖已安装应用）**：已安装应用有新版本时，覆盖安装正常完成，数据不丢失
- [ ] **降级安装被拦截**：尝试安装低版本时系统拒绝，客户端状态正确回退

## 4. 升级链路回归

### 4.1 升级检测

- [ ] **检测单个应用有升级**：已安装应用有新版本时，详情页或"我的应用"中显示升级标识
- [ ] **检测全部应用升级**：调用 checkAllUpgrades 后，所有有升级的应用均被正确标记
- [ ] **版本号相同时不触发升级**：已安装版本与最新版本相同时，hasUpgrade 为 false

### 4.2 升级执行

- [ ] **单个应用升级成功**：点击升级后，下载-校验-安装全链路顺利完成
- [ ] **批量串行升级成功**：选择多个应用批量升级，依次串行执行，全部完成
- [ ] **批量升级中某个失败时停止后续**：队列中某个应用升级失败，后续应用不再执行，已完成的保持成功状态
- [ ] **升级失败后可重试**：失败的应用可单独或重新批量触发升级

## 5. 策略拦截回归

### 5.1 网络策略

- [ ] **非 Wi-Fi 时下载被拦截**：断开 Wi-Fi 仅保留移动网络时，发起下载弹出拦截提示
- [ ] **Wi-Fi 恢复后拦截自动解除**：重新连接 Wi-Fi 后，可正常发起下载

### 5.2 存储策略

- [ ] **存储空间不足时安装被拦截**：可用空间不足时，安装操作被拦截并提示

### 5.3 车况策略

- [ ] **行车状态下安装被拦截（需 OEM 信号）**：行车状态时安装操作被拦截，提示"行车中无法安装"（需要 OEM 信号已接入）
- [ ] **驻车后策略自动刷新**：从行车切换到驻车后，被拦截的操作可正常发起

### 5.4 策略提示

- [ ] **策略拦截后错误信息正确展示**：拦截原因文案清晰（如"当前非 Wi-Fi 网络"、"存储空间不足"、"行车中暂不可操作"），并提供可操作的后续引导

## 6. 状态与页面回归

### 6.1 首页与搜索

- [ ] **首页应用列表加载**：应用列表正常展示，包含应用图标、名称、标签、简介
- [ ] **搜索功能正常**：输入关键词后搜索结果匹配，结果列表展示完整
- [ ] **搜索关键词高亮或匹配验证**：搜索结果与输入关键词相关

### 6.2 详情页

- [ ] **详情页展示正确**：应用详情页包含完整信息（描述、版本、大小、权限、评分、开发者）
- [ ] **详情页主操作按钮状态正确**：未安装显示"下载"、下载中显示"暂停"、下载完成显示"安装"、已安装显示"打开"、有升级显示"升级"

### 6.3 我的应用

- [ ] **我的应用列表展示**：已安装应用正确列出，与系统 PackageManager 一致
- [ ] **我的应用升级标识**：有升级的应用显示升级角标

### 6.4 任务中心

- [ ] **下载中心任务列表**：下载中、已暂停、已完成的任务均正确展示
- [ ] **安装中心任务列表**：安装中、等待确认、已完成、已失败的任务均正确展示
- [ ] **升级中心任务列表**：待升级、升级中、已完成、已失败的升级任务均正确展示

### 6.5 实时状态

- [ ] **下载过程中进度条实时更新**：进度百分比平滑递增，与实际下载字节对应
- [ ] **安装确认等待状态正确展示**：系统确认页弹出期间，任务状态显示"等待用户确认"
- [ ] **错误信息可读且可操作**：错误状态下展示错误原因，并提供"重试"或"取消"操作

## 7. 异常场景回归

### 7.1 进程级异常

- [ ] **强杀进程后冷启动状态恢复**：下载中/安装中/升级中的任务，强杀后重新打开状态不丢失
- [ ] **长时间后台运行后回到前台状态正确**：应用切到后台运行超过 30 分钟后切回前台，任务列表和状态正确

### 7.2 操作级异常

- [ ] **快速连续点击不产生重复任务**：对同一应用快速点击下载/安装按钮 5 次以上，仅产生一个任务
- [ ] **同一应用同时下载和安装不冲突**：极端情况下不会同时出现下载和安装两个任务

### 7.3 系统级异常

- [ ] **低内存场景下应用不崩溃**：通过开发者选项限制后台进程后，应用主流程不崩溃
- [ ] **存储空间在下载过程中耗尽**：下载过程中可用空间降为 0，任务标记失败并提示，应用不崩溃
- [ ] **系统时间被篡改**：设备时间设置到过去或未来，不影响本地缓存的正常读取
