# M17 字符串资源治理与 R1 文档继续归并

## 1. 背景
在 M16 完成文档归并、依赖集中和 Java/Kotlin 11 收平后，工程里仍存在两类残留：

1. 部分字符串资源 id 仍是不可维护的旧转义式命名
2. R1 相关文档仍拆成多份，不利于接手

本轮目标就是把这两块继续收口。

---

## 2. 本轮完成项

### 2.1 字符串资源命名治理
已将 `common` 中残留的异常资源名统一改为可读命名，例如：

- `ui_service_center_subtitle`
- `ui_home_recommend_recent_quick_entry`
- `ui_task_center_header_hint`
- `ui_download_preferences_summary`
- `ui_install_controls_summary_zero`
- `ui_upgrade_controls_summary_zero`
- `ui_download_environment_description`

同时，相关布局引用已同步更新：

- `common/src/main/res/layout/view_task_center_*.xml`
- `feature-home`
- `feature-search`
- `feature-detail`
- `feature-myapp`
- `feature-downloadmanager`
- `feature-installcenter`
- `feature-upgrade`
- `feature-debug`

### 2.2 app 资源归位
`app` 模块不再重复持有整套 UI 文案。

当前策略：

- 公共 UI 文案统一由 `common` 提供
- `app/src/main/res/layout/activity_main.xml` 改为直接引用 `common` 资源命名空间
- `app/src/main/res/values/strings.xml` 仅保留 app 自身必须资源

### 2.3 脚手架一致性修正
`setup_project.py` 已同步对齐当前工程基线：

- Java / Kotlin 目标改为 11
- 增加 `gradle/libs.versions.toml` 生成内容
- 生成的 `app/build.gradle.kts` 改为 version catalog 依赖写法

### 2.4 R1 文档继续归并
原先这几份文档：

- `18-R1真实下载器设计稿.md`
- `19-R1收口文档.md`
- `20-真实下载源接入说明.md`
- `21-DownloadEnvironment可切换配置入口.md`
- `22-开发者设置页与环境切换.md`

已合并为：

- `18-R1真实下载器与下载环境治理.md`

现在下载器设计、下载源治理、环境 provider、开发设置页切换入口，统一从这一份文档进入。

---

## 3. 当前结果
经过 M17 后，工程在治理层面的状态变成：

- Java / Kotlin 11 基线统一
- 依赖统一到 `gradle/libs.versions.toml`
- 公共 UI 文案统一由 `common` 承载
- 不再允许引入新的异常长转义式字符串资源 id
- R1 下载器与下载环境文档收束为单一入口

---

## 4. 当前仍未完成
仍未完成的关键点没有变化：

- 仓库没有 `gradlew`
- 环境没有系统 `gradle`
- 因此仍无法做真实 Gradle 编译验收
