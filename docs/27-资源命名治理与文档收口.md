# 27. 资源命名治理与文档收口

## 1. 当前结论

这份文档总结的是资源命名治理和文档收口阶段留下来的长期规则。  
其中两件事今天仍然成立：

- 字符串资源命名治理
- R1 下载相关文档归并

而且现在这两件事都已经被继续向前推进：

- 资源命名治理成为当前持续约束
- 下载、安装、升级、状态、策略、管理等文档口径已经进一步统一

## 2. M17 当时完成了什么

### 字符串资源命名治理

- 清理旧式、不可维护的资源 id
- feature / common / app 引用同步改名

### app 资源归位

完成了：

- `app` 不再重复持有整套公共 UI 文案
- 公共资源更多由 `common` 承载

### R1 文档归并

完成了：

- 下载器设计、下载源治理、环境切换入口统一收口到 `18-` 文档

## 3. 今天的延续价值

从今天看，M17 最大的价值有两点：

1. 它让资源命名治理不再是一次性清理，而变成长期规范
2. 它证明了“文档应该归并成稳定入口”，这也直接支撑了后面对 `01-27` 的继续重梳

## 4. 当前仍需保持的纪律

- 新增字符串资源继续使用可读、可维护命名
- 动态模板不要重新散回页面代码里硬编码
- 文档继续保持单主题单入口，避免重新碎片化

## 5. 一句话总结

M17 的成果已经从“阶段治理动作”变成了当前工程的长期规范：资源名要可读，文档入口要统一。
