# 02. 项目目录结构与模块说明

## 2. 主要代码目录说明

### 2.1 app/
负责应用入口与全局装配：
- `App.kt`：Application 入口
- `AppContainer.kt`：手动依赖注入容器
- `MainActivity.kt`：主页面容器
- `MainNavigator.kt`：页面跳转接口

### 2.2 common/
存放项目通用层：
- `base/`：BaseFragment / BaseViewModel
- `result/`：统一结果和错误定义
- `ui/`：统一 UI 样式和格式化工具

### 2.3 core/
基础能力层：
- `downloader/`：下载器抽象与实现
- `installer/`：安装器抽象与实现
- `logger/`：日志
- `tracker/`：打点

### 2.4 data/
数据聚合层：
- `model/`：数据模型
- `datasource/`：本地/远端/系统数据源
- `repository/`：统一数据访问出口

### 2.5 domain/
业务层，承载 7 个核心模块：
- `download/`
- `install/`
- `upgrade/`
- `appmanager/`
- `state/`
- `policy/`
- `repository` 的接口能力由 `data/repository` 承接

### 2.6 feature/
页面级 MVVM：
- `home/`
- `search/`
- `detail/`
- `myapp/`
- `downloadmanager/`
- `upgrade/`

## 3. 页面与业务模块对应关系

| 页面 | 主要依赖模块 | 说明 |
|---|---|---|
| 首页 | 应用管理模块、状态中心、策略中心 | 展示应用列表、状态、策略提示 |
| 搜索页 | 应用管理模块、状态中心、策略中心 | 搜索应用并展示策略提示 |
| 详情页 | 下载模块、安装模块、升级模块、状态中心 | 执行下载/安装/升级/打开 |
| 我的应用 | 应用管理模块、状态中心 | 展示已安装和任务关联状态 |
| 下载管理页 | 下载模块、安装模块、状态中心、策略中心 | 任务中心、批量操作、策略控制 |
| 升级管理页 | 升级模块、状态中心 | 升级任务中心 |

## 4. 模块化分层原则

### UI 层不越界
UI 只通过 ViewModel 调用业务层，不直接操作数据源。

### 业务层负责编排
业务层负责流程控制和状态更新，不直接处理 UI 表现。

### 数据层提供统一数据
Repository 屏蔽 remote / local / system 的差异。

## 5. 当前整理建议

后续继续演进时，建议保持：
- **功能增量优先加在 domain / data / core**
- **页面只做轻量展示和交互**
- **新流程优先补时序图，再补实现**
