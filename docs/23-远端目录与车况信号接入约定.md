# 23. 远端目录与车况信号接入约定

## 1. 目的

本文档用于说明当前工程为“真实后端目录接入”和“真实车况/OEM 信号接入”预留的接入点。

当前仓库已经完成：

- 商店目录 `HTTP -> 本地缓存 -> 资源目录` 回退链路
- 目录 DTO 与 HTTP client 抽象
- 策略中心的实时信号提供者抽象

因此后续接真实系统时，不应直接改页面或业务编排，而应优先替换 provider / client。

---

## 2. 远端目录接入点

### 2.1 当前结构

- `AppCatalogHttpClient`
- `HttpUrlConnectionAppCatalogHttpClient`
- `AppCatalogResponse`
- `AppCatalogItemResponse`
- `ResilientAppCatalogSource`
- `AppRemoteDataSource`

### 2.2 当前回退顺序

1. HTTP 拉取目录
2. 本地缓存目录
3. `raw/app_store_catalog.json`

### 2.3 后端接入建议

优先保持接口字段与 `AppCatalogItemResponse` 对齐：

- `appId`
- `packageName`
- `name`
- `description`
- `versionName`
- `category`
- `editorialTag`
- `recommendedReason`
- `searchKeywords`
- `developerName`
- `ratingText`
- `sizeText`
- `lastUpdatedText`
- `compatibilitySummary`
- `permissionsSummary`
- `updateSummary`
- `latestVersion`
- `hasUpgrade`
- `changelog`

### 2.4 不建议的做法

- 不要在 `Fragment` / `ViewModel` 里直接请求目录接口
- 不要把接口字段映射逻辑散到 `AppManager`
- 不要绕过 `AppRemoteDataSource`

---

## 3. 车况/OEM 信号接入点

### 3.1 当前结构

- `PolicyRuntimeSignalProvider`
- `AndroidPolicyRuntimeSignalProvider`
- `PolicyRuntimeSignals`
- `DefaultPolicyCenter`

### 3.2 当前已接入信号

- Wi‑Fi 状态
- 低存储状态

### 3.3 当前未接入信号

- 真实驻车状态
- OEM 自定义车况
- 点火状态 / 电源状态
- 更细的前后台限制

### 3.4 OEM 接入建议

后续若车机平台能提供驻车或驾驶状态：

1. 新增一个实现 `PolicyRuntimeSignalProvider` 的 OEM provider
2. 在 `AppContainer` 中替换 `AndroidPolicyRuntimeSignalProvider`
3. 保持 `DefaultPolicyCenter` 不变

### 3.5 不建议的做法

- 不要把 OEM 广播解析直接写进 `DefaultPolicyCenter`
- 不要让页面自己判断车况限制
- 不要让下载/安装/升级模块各自监听系统广播

---

## 4. 一句话总结

真实后端和 OEM 接入的推荐路径是：

**替换目录 client / 替换实时信号 provider，而不是改业务层。**
