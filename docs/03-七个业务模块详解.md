# 03. 七个业务模块详解

## 1. 下载模块

### 职责
- 创建下载任务
- 暂停 / 继续 / 取消 / 重试
- 驱动 FileDownloader
- 更新下载状态
- 持久化任务与文件路径
- 冷启动恢复下载任务

### 不负责
- 页面展示
- 安装执行
- 升级编排

### 关键输入
- appId
- 下载地址（当前为模拟）
- 用户操作
- 策略中心结果

### 关键输出
- 下载状态
- 下载进度
- 文件路径
- 失败码/失败信息

---

## 2. 安装模块

### 职责
- 发起安装
- 校验 apk 文件
- 驱动 PackageInstaller
- 更新安装状态
- 写入已安装结果

### 不负责
- 下载流程
- 升级版本判断

---

## 3. 升级模块

### 职责
- 检查版本
- 判断是否可升级
- 编排 下载 + 安装
- 维护升级状态
- 支持批量开始/批量重试

### 不负责
- 具体下载实现
- 具体安装实现

---

## 4. 应用管理模块

### 职责
- 聚合首页应用卡片
- 聚合搜索结果
- 聚合我的应用
- 聚合下载任务视图
- 聚合安装任务视图
- 聚合升级任务视图
- 提供任务统计和策略提示

### 定位
它是面向“应用实体”和“任务视图”的聚合层。

---

## 5. 状态中心

### 职责
- 统一维护下载状态
- 统一维护安装状态
- 统一维护升级状态
- 提供 Flow / 观察能力
- 为按钮态和标签态提供依据

### 关键意义
它是 UI 和业务流程之间的统一状态源。

---

## 6. 策略中心

### 职责
- 判断是否允许下载
- 判断是否允许安装
- 判断是否允许升级
- 管理网络状态（Wi‑Fi / 蜂窝）
- 管理车况（驻车 / 行车）
- 管理存储状态（正常 / 不足）

### 当前策略示例
- 非 Wi‑Fi 时禁止下载
- 行车时禁止安装
- 存储不足时禁止下载和安装

---

## 7. Repository

### 职责
- 聚合 remote / local / system 数据
- 提供应用详情 / 升级信息 / 已安装信息
- 提供任务持久化
- 提供偏好持久化

### 定位
它是 7 个模块中的“统一数据访问入口”。

## 8. 七个模块边界总结

| 模块 | 核心关键词 | 一句话说明 |
|---|---|---|
| 下载模块 | download task | 只管下载流程与任务状态 |
| 安装模块 | install task | 只管安装流程与安装结果 |
| 升级模块 | upgrade orchestration | 只管升级编排 |
| 应用管理模块 | app aggregation | 面向页面提供统一应用/任务视图 |
| 状态中心 | single source of truth | 全局统一状态源 |
| 策略中心 | policy decision | 决策是否允许执行 |
| Repository | data aggregation | 统一数据出口 |
