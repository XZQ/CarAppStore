# 车载应用商店项目进度交接文档（最新：R1 / R2 / D 收口 + M1 ~ M15 收口 + M16 / M17 工程治理）

## 1. 项目定位
这是一个车载应用商店 Android 项目，采用 MVVM 架构。

当前项目已经完成：

- 主干页面齐全
- 任务中心公共基座完成
- R1 真实下载器原型、下载源治理、环境切换入口收口
- R2 真实安装器原型收口
- D 统一数据层第一轮收口
- M1 ~ M15 多 module 与 feature module 第一轮收口
- M16 / M17 两轮工程治理：文档归并、依赖集中管理、编译目标统一、字符串资源命名治理

---

## 2. 当前必须遵守的约束
后续任何会话都必须保持这些约束不变：

- 采用 MVVM
- 不使用 Hilt
- 不使用 Navigation
- 页面跳转使用 `Activity + FragmentManager`
- 依赖注入使用手动依赖注入 / `AppContainer`
- 包名统一为 `com.nio.appstore`

---

## 3. 当前 module 结构
当前已经建立并在使用的 module：

```text
:app
:common
:core
:data
:business
:feature-home
:feature-detail
:feature-myapp
:feature-search
:feature-downloadmanager
:feature-upgrade
:feature-installcenter
:feature-debug
```

当前依赖方向：

- `:core -> :common`
- `:data -> :common + :core`
- `:business -> :common + :core + :data`
- `:feature-* -> :common + :core + :data + :business`
- `:app -> :common + :core + :data + :business + feature-*`

## 4. 当前工程基线

### 4.1 Java / Kotlin 版本
当前所有 module 已统一为：

- `sourceCompatibility = JavaVersion.VERSION_11`
- `targetCompatibility = JavaVersion.VERSION_11`
- `jvmTarget = "11"`

### 4.2 依赖管理
当前三方库依赖已集中到：

- `gradle/libs.versions.toml`

各 module `build.gradle.kts` 已改为通过 version catalog 引用依赖，不再在每个 module 内重复写版本号。

### 4.3 字符串与资源治理
当前公共 UI 文案统一由 `common` 提供，`app` 不再重复持有整套 UI 文案。

已经完成：

- `common` 中残留的旧转义式异常资源名改为可读 id
- feature / common / app 布局同步改用新 id
- `app/src/main/res/layout/activity_main.xml` 直接引用 `common` 资源命名空间
- `app/src/main/res/values/strings.xml` 收缩为 app 自身必须资源

后续要求：

- 新增字符串必须使用可读、可维护的命名
- 不允许继续引入异常长的转义式字符串资源 id

### 4.4 脚手架一致性
`setup_project.py` 已同步对齐当前工程基线：

- Java / Kotlin 11
- version catalog 依赖写法
