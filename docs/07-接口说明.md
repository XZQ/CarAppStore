# 接口说明

本文整理当前工程里最关键的接口与职责，方便团队在继续实现真实下载器、真实安装器、远端数据源时保持边界一致。

## 1. AppManager
作用：面向页面输出聚合后的视图模型，不直接承载底层下载或安装细节。

当前关键能力：
- `getHomeApps()`：首页应用卡片
- `getMyApps()`：我的应用聚合视图
- `searchApps(keyword)`：搜索结果
- `getDownloadTasks()`：下载任务中心
- `getInstallTasks()`：安装任务中心
- `getUpgradeTasks()`：升级任务中心
- `getDownloadTaskStats()` / `getInstallTaskStats()` / `getUpgradeTaskStats()`：统计数据
- `getPolicyPrompt()`：当前策略提示文案

## 2. DownloadManager
作用：下载流程编排层。

职责：
- 创建下载任务
- 暂停 / 恢复 / 删除任务
- 与 `FileDownloader` 协作完成文件落盘
- 更新 `StateCenter`
- 任务持久化与恢复

## 3. InstallManager
作用：安装流程编排层。

职责：
- 发起安装
- 安装前本地包校验
- 与 `PackageInstaller` 协作完成安装流程
- 安装结果回写 `StateCenter`

## 4. UpgradeManager
作用：升级流程编排层。

职责：
- 查询升级信息
- 判断升级可执行性
- 调下载模块
- 调安装模块
- 统一回写升级状态

## 5. StateCenter
作用：全局统一状态源。

维护内容：
- `DownloadStatus`
- `InstallStatus`
- `UpgradeStatus`
- `PrimaryAction`
- 进度、错误信息、错误码、本地安装包路径、安装版本

## 6. PolicyCenter
作用：车机场景策略判断。

当前已经接入：
- Wi‑Fi / 蜂窝
- 驻车 / 行车
- 存储正常 / 不足

## 7. FileDownloader / PackageInstaller
这两个接口是当前最重要的“底层可替换点”。

### FileDownloader
当前实现：`SimulatedFileDownloader`
后续可以替换成：
- OkHttp 下载器
- 断点续传下载器
- 后台 Service 下载器

### PackageInstaller
当前实现：`SimulatedPackageInstaller`
后续可以替换成：
- 系统安装适配层
- 安装会话模式
- OEM 安装桥接层
